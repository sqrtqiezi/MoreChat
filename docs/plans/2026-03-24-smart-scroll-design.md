# 会话窗口智能滚动设计

Issue: #8 - 会话窗口信息消息接收时，增强使用友好性

## 问题

当前收到新消息时无条件滚动到底部，导致：
1. 用户正在 lightbox 查看图片时被关闭
2. 用户浏览历史消息时被中断

## 方案：条件性滚动 + 新消息浮动提示

### 滚动行为

- 会话切换 → 无条件滚动到底部
- 新消息到达 + 用户在底部 → 自动滚动
- 新消息到达 + 用户不在底部 → 不滚动，显示浮动提示

### 新消息计数

- `unreadCount` 状态：用户不在底部时每收到新消息 +1
- 重置时机：滚动回底部、点击提示按钮、切换会话

### 浮动按钮

- 位置：消息列表底部居中
- 内容：`↓ N 条新消息`
- 点击：滚动到底部 + 重置计数
- 显示条件：`unreadCount > 0 && !isAtBottom`

### 涉及文件

| 文件 | 改动 |
|------|------|
| `useMessageScroll.ts` | 条件性滚动，用 ref 跟踪底部状态 |
| `useMessages.ts` | 新增 `unreadCount` 和重置方法 |
| `MessageList.tsx` | 集成浮动提示组件 |
| `NewMessageIndicator.tsx`（新建） | 浮动按钮组件 |
