# MoreChat 端到端测试计划

## 测试目标

基于生产环境的真实数据，验证 MoreChat 的核心功能和用户流程，确保：
1. 所有关键用户路径正常工作
2. 前端展示与后端 API 数据一致
3. 实时功能（WebSocket）正常运行
4. 错误场景得到正确处理

## 测试环境

- **测试环境**: 生产服务器 `http://101.132.162.209:8079`
- **测试工具**: Playwright
- **测试数据**: 从生产环境抓取的真实数据
- **浏览器**: Chromium (Desktop Chrome)

## 测试数据准备

### 已准备
- ✅ 10条最新消息样本
- ✅ 5个会话样本
- ✅ 5个联系人样本

### 待补充

## 测试用例清单

### 1. 认证与登录 (Authentication)

#### 1.1 成功登录
- **优先级**: 🔴 P0
- **前置条件**: 无
- **测试步骤**:
  1. 访问首页 `/`
  2. 自动重定向到 `/login`
  3. 输入正确密码
  4. 按 Enter 或点击登录按钮
- **预期结果**:
  - 登录成功，重定向到 `/` (Knowledge Page)
  - localStorage 中保存 token
  - 可以看到用户昵称

#### 1.2 登录失败
- **优先级**: 🟡 P1
- **测试步骤**:
  1. 访问 `/login`
  2. 输入错误密码
  3. 提交
- **预期结果**:
  - 显示错误提示 "密码错误"
  - 停留在登录页面

#### 1.3 Token 过期处理
- **优先级**: 🟡 P1
- **测试步骤**:
  1. 使用过期的 token
  2. 访问任意受保护页面
- **预期结果**:
  - 自动重定向到 `/login`
